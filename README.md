# 呼吸机 PCV 模式高精度闭环控制系统 (Ventilator PCV Control System)

本项目基于 Nuvoton M463 微控制器平台开发，致力于解决医用呼吸机在 **PCV（压力控制通气，Pressure Control Ventilation）模式** 下压力超调、气流顿挫及高频机械震荡等控制难题。

通过引入经过深度优化的**增量式 PID 控制算法 (Incremental PID)**，全面接管比例阀 (Proportional Valve, PVQ33) 的吸气控制，实现了极其平滑的流体力学充气波形与精准的平顶期压力维持。

---

## 🌟 核心特性 (Key Features)

### 1. 增量式 PID 闭环算法
相比于传统的位置式 PID，本项目采用增量式 PID，具备天然的流体动态控制优势：
- **极致安全防飞车**：算法仅计算控制量的“增量(Δu)”，彻底消灭了积分过度累积（Windup）导致的阀门瞬间满量程“大开”风险，极大降低了患者气压伤风险。
- **天然阻尼防超调**：增量式中的 P 项对误差变化率起作用（充当微分阻尼），在气压快速上升时能提前介入强大的“刹车”力矩。
- **无扰动切换 (Bumpless Transfer)**：算法能够在开环与闭环模式（如 CPAP 到 PCV）之间实现丝滑过渡，没有控制量突跳。

### 2. 机械限幅低通滤波 (Slew Rate Limiting)
针对硬件比例阀固有的响应滞后与死区（Deadband）问题，在 PID 输出端引入了严格的**对称变化率限幅 (`max_delta = 10.0f`)**。
该限幅机制充当了绝佳的“机械低通滤波器”，强制过滤掉传感器白噪声带来的微小抖动，从根本上消灭了阀门高频打颤（Limit Cycle）现象，使得气流曲线如丝般平滑。

### 3. 目标偏移补偿黑科技 (Target Offset)
为了打破“控制平滑度”与“气路惯性超调”之间的死锁矛盾，系统创新性地引入了设定值补偿机制。
在不破坏底层平滑对称算法的前提下，底层代码自动扣除固定物理惯性余量（如 `1.2 cmH2O`）。通过“欺骗”PID 去追击较低的内部目标，依靠流体惯性刚好完美滑行并精准停在真实设定值上（0超调）。

### 4. 显控分离架构 (Display-Control Separation)
- **底层硬件控制（0延迟）**：PID 控制器直接挂载在 5ms 的底层中断中，读取无滤波、零延迟的原始传感器 ADC 数据，保证最高反应帧率，杜绝因滤波相移导致的震荡。
- **上层 UI 显示（高平滑）**：在屏幕重绘函数（`draw()`）中剥离出独立的 IIR 一阶低通滤波器，过滤显示噪点。做到了“底层控制快准狠，上层界面稳准美”。

## 🛠️ 系统架构与核心文件 (Architecture)

* `main.c`: 包含系统的初始化、5ms 主控定时器中断、传感器数据读取以及屏幕 UI 波形绘制（含显示端独立滤波器）。
* `pcv_pid_controller.c` / `pcv_pid_controller.h`: 
  核心增量式 PID 算法库。包含状态重置 `PCV_Reset()`、死区补偿算法以及控制量更新循环 `PCV_PID_Update()`。

---

## ⚙️ 硬件与开发环境 (Environment)

- **MCU**: Nuvoton M463 系列 ARM Cortex-M4 微控制器
- **IDE / 编译器**: Keil MDK-ARM (uVision)
- **执行机构**: SMC PVQ33 系列比例电磁阀
- **控制周期**: 5ms (200Hz)

## 💡 开发与调参经验
在呼吸机开发中，“气流顺畅度”的优先级往往高于“绝对的数学收敛速度”。
如果遇到稳态误差或惯性超调，**优先使用目标偏移（Target Offset）或前馈（Feed-Forward）机制**，而不要随意解除限幅或使用极不对称的刹车参数，否则会严重破坏流体在管路中的连续性，引起患者不适甚至人机对抗。
